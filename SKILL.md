# 软件著作权代理技能

用于两类任务：
1. 制作软著申请材料
2. 审核已有软著申请材料

## 触发条件

用户提到以下任一场景时启用本技能：
- 申请软著 / 软著申请 / 软件著作权 / 软件申请
- 帮我整理软著材料
- 帮我审核软著材料 / 看看材料是否合规

## 内置脚本（必须优先复用）

程序鉴别材料的 **md / html / pdf 三件套** 不得手写拼装或另写脚本，应直接调用本 skill 的 `scripts/` 工具（排版与 [assets/code-pages-60.html](assets/code-pages-60.html) 样例一致）：

| 文件 | 作用 |
|------|------|
| [scripts/generate_program_pdf.py](scripts/generate_program_pdf.py) | **主入口**：按「用户源码优先」规则扫描并过滤，生成 `程序鉴别材料.md`、`code-pages-60.html`、`程序鉴别材料.pdf` |
| [scripts/程序鉴别材料-转PDF.bat](scripts/程序鉴别材料-转PDF.bat) | Windows 快捷方式，内部调用上述 Python 脚本 |
| [assets/code-pages-60.html](assets/code-pages-60.html) | **排版参考样例**（skill 内静态样例）；正式 60 页材料写入目标项目的 `software-copyright/` |

执行即固定生成 **60 页**（前 30 + 后 30，每页 50 行代码），无页数参数：

```bash
python "%USERPROFILE%\.cursor\skills\software-copyright-skill\scripts\generate_program_pdf.py" ^
  --project-root "<项目根目录>" ^
  --project-name "<与基本信息一致的软件全称>" ^
  --version "V1.0"
```

或：

```bat
"%USERPROFILE%\.cursor\skills\software-copyright-skill\scripts\程序鉴别材料-转PDF.bat" "<项目根目录>" "<软件全称>" "V1.0"
```

依赖：`pip install reportlab`（PDF 用 ReportLab + 系统黑体，勿改用 wkhtmltopdf 另起一套逻辑）。

输出一律写入 `<项目根>/software-copyright/`，**不要**覆盖 skill 目录内 `assets/code-pages-60.html` 模板文件。

默认仅按本文件执行。遇到以下情况再读取详细资料：
- [references/reference.md](references/reference.md)：完整字段清单、字数限制、枚举

- [references/review-checklist.md](references/review-checklist.md)：**审核/审查模式**完整清单（实名认证、附加证明、源码与操作文档）

## 工作原则（必须遵守）

1. 先自动采集，再提问；只问“必填且无法推断”的项目。  
2. 所有字段都做长度校验，不合规时给出压缩建议。  
3. 软件名称全链路一致：申请表、代码页眉、文档页眉必须完全一致。  
4. 输出要可直接提交或可直接执行补正，不输出空泛建议。  
5. 若信息不足，明确列出“缺失项 + 最小提问列表”。  
6. 硬件环境字段必须包含基础参数：CPU、内存、硬盘、显卡；**不需要**填写显示器、屏幕尺寸、分辨率等显示外设信息。
7. 《说明文档》页数硬性要求：**超过 7 页**（不含 7 页），保证内容丰富和完整，**不超过 60 页**；生成后须按 PDF 页数自检，不足则扩充模块说明与截图，超出则精简。

## 信息采集顺序

按顺序扫描并提取：
- `package.json` / `package-lock.json`
- `pom.xml` / `build.gradle*`
- `requirements.txt` / `pyproject.toml`
- `README.md`
- 项目目录结构与源码扩展名（识别语言与规模）
- Docker/部署配置（推断运行环境）

自动推断项：
- 开发语言（可多选）
- 代码量（源程序量）
- 开发环境与运行平台候选值
- 技术特点候选标签

## 字段与限制

### 基本信息

- 软件全称：建议格式 `XXX软件` / `XXX系统` / `XXX平台`
- 开发硬件环境：<=50字，必须包含 CPU/内存/硬盘/显卡；**不要**写显示器、屏幕、分辨率
- 运行硬件环境：<=50字，必须包含 CPU/内存/硬盘/显卡；**不要**写显示器、屏幕、分辨率
- 开发操作系统：<=50字
- 开发工具/环境：<=50字
- 运行平台/操作系统：<=50字
- 运行支撑环境/支持软件：<=50字
- 自定义编程语言描述：<=120字
- 开发目的：<=50字
- 面向领域/行业：<=50字
- 软件主要功能：500-1300字
- 自定义技术特点描述：<=100字

### 编程语言（标准候选）

Assembly language, C, C#, C++, Delphi/Object Pascal, Go, HTML, Java, JavaScript, MATLAB, Objective-C, PHP, PL/SQL, Perl, Python, R, Ruby, SQL, Swift, Visual Basic, Visual Basic .Net

### 技术特点（候选）

APP、游戏软件、教育软件、金融软件、医疗软件、地理信息软件、云计算软件、信息安全软件、大数据软件、人工智能软件、VR软件、5G软件、小程序、物联网软件、智慧城市软件

## 材料标准

### 程序鉴别材料

- 交存方式：一般交存 / 例外交存
- 默认要求：连续前30页 + 连续后30页
- 每页不少于50行
- 若源码不足60页：提交全部源码
- PDF，建议 <=50MB
- **源码采集优先级（必须遵守）**：
  - **优先**收录用户编写的业务源程序（如 `src/`、`app/`、`pages/`、`components/`、`api/`、`services/` 等目录下的 `.vue`、`.js`、`.ts`、`.py`、`.java` 等源文件）。
  - **禁止**纳入：用户配置文件（`package.json`、`tsconfig.json`、`vite.config.*`、`.env`、`pages.json` 等）、数据与锁文件（`*.lock`、`requirements.txt`、数据集目录）、IDE/构建/依赖产物（`node_modules/`、`dist/`、`unpackage/`、`.idea/`、`.vscode/`、`__pycache__/`、生成代码目录等）。
  - 测试、示例、文档目录（`test/`、`__tests__/`、`examples/`、`docs/` 等）仅在前 60 页仍不足时作为**最后**补充；Agent 调用 `generate_program_pdf.py` 时脚本已按此规则过滤与排序，不得手工混入上述文件。
- 生成格式规则（必须遵守）：
  - 固定 **60 页**（连续前 30 + 连续后 30），第 1–59 页每页 **50 行**有效代码，第 60 页 **至少 40 行**且以**完整模块**结尾（不得在模块中间截断）
  - **页眉**：居中为「软件全称+版本号」（如 `某某管理系统V1.0`，与基本信息一致）；**右上角**为页码（`第 N 页`）
  - **不输出任何行号**（无源文件行号、无页内 01~50 前缀）
  - 行格式：`文件名:  代码`（冒号后固定两个空格，再接代码）
  - 左侧文件名长度 <= 30 字符（超出时保留尾部并前置省略号）
  - 代码行左侧固定显示文件名（不显示目录路径）
  - 每个文件首行前插入：`模块:<文件夹路径>`
  - `模块:` 行为分组标识，不计入“每页50行代码”统计
  - `程序鉴别材料.md`、`code-pages-60.html`、`程序鉴别材料.pdf` 三份内容必须同源一致
  - 文档层出现的软件名称（标题、页眉、说明）必须与“软件全称”完全一致；版本号与 `--version`（默认 `V1.0`）一致
  - 作为模板输出时，不得泄露真实项目名称；历史名称（如“艾宾浩斯背诗工具”）应在生成阶段统一替换为 `PROJECT_NAME`
  - 生成程序鉴别材料时应移除注释（单行注释、块注释、HTML注释），保留非注释代码内容

### 文档鉴别材料

**必须**基于 skill 内置模板生成，结构与 [assets/说明文档模板.doc](assets/说明文档模板.doc) / [assets/说明文档模板.md](assets/说明文档模板.md) 一致；详细规则见 [references/doc-template.md](references/doc-template.md)。

- 文档类型：《说明文档》（用户手册/操作说明），**图文结合**
- **页数（硬性，必须遵守）**：《说明文档》内容需**超过 7 页**（第 8 页起才算达标），保证内容丰富和完整，**不超过 60 页**
  - 导出 PDF 后自检：页数 ≤7 不合格，须增加功能模块说明与截图后再导出；页数 >60 须精简
  - 实务上 13–20 页较易通过审核，但最低门槛仍是 **>7 且 ≤60**
- 页眉：居中「软件全称+版本号」；**右上角**页码「第 N 页」
- **标题**：`文档鉴别材料.pdf` 文首**软件全称标题须居中**（与模板一致；导出 Word/PDF 时设置标题段落居中对齐）
- 截图：清晰完整、流程连贯；**有登录界面时从登录开始**；无登录可省略登录章节；无界面截图时提供流程图
- **图注（必须）**：每张截图**下方居中**标注，格式 `图N-简述`（如 `图1-首页`、`图2-登录`）；全文从 1 连续编号，简述与对应功能/界面一致
- 正式提交前**删除**模板文首「注意事项」等说明性段落
- 结构（固定）：
  - **第一章 系统描述**：1.1 系统价值、1.2 系统功能描述
  - **第二章 系统模块介绍**：登录（如有）→ 主界面 → 各核心功能（功能名 + 文字说明 + 截图 + **图注**）
- 输出：`software-copyright/文档鉴别材料.md`（按模板填充）+ `screenshots/` 截图目录 + 最终 `文档鉴别材料.pdf`
- PDF，建议 <=50MB

### 其他证明文件

审核时按 [references/review-checklist.md](references/review-checklist.md) 判断下列材料是否**必须提交**（符合情形则缺一不可）：

| 材料 | 触发情形（摘要） |
|------|------------------|
| **实名认证** | 提交前须在版权中心完成：<https://register.ccopyright.com.cn/login.html> |
| 合作开发协议 | 多人著作权人 + 合作开发 |
| 委托开发协议 | 开发方式为委托开发 |
| 前期开发说明 | 成立与完成日期不足 1 个月；或不足半年且程序量较大（约 1 万行/月） |
| 非职务开发声明 | 企业/事业单位/机关与自然人合作开发（全自然人不需要） |
| 上级法人单位证明 | 著作权人为分公司 |

- PDF，建议 <=50MB，可多个

## 执行流程

### A. 生成模式（用户要“做材料”）

1. 自动扫描项目并给出“已识别信息草稿”。
2. 仅询问缺失必填项（一次性列出，避免反复追问）。
3. 生成材料文案与文件清单；**程序鉴别材料** 在确认软件全称后，调用 `scripts/generate_program_pdf.py`（见上文「内置脚本」），勿重复实现。**文档鉴别材料** 须读取 [assets/说明文档模板.md](assets/说明文档模板.md) 按模板结构生成，并列出待补截图清单；**导出 PDF 后确认页数 >7 且 ≤60**。
4. 校验长度与一致性；不合规则自动改写并标注改动原因。
5. 输出目录与下一步提交指引。

### B. 审核模式（用户要“审材料”）

1. **先读** [references/review-checklist.md](references/review-checklist.md)，确认实名认证与附加证明是否触发。
2. 接收用户材料（或材料清单），按审查清单逐项审核（完整性、签章、格式、一致性、截图覆盖）。
3. 生成问题清单（高/中/低优先级）；附加证明缺失标为 **[高]**。
4. 给出逐项补正建议和可直接替换文本。
5. 输出复审结论：通过 / 需修改 / 不通过。

## 询问规范（最小提问）

禁止询问可自动识别的信息（语言、代码量、部分环境字段）。  
每次提问都要附：
- 为什么要问（该字段是哪个必填项）
- 字数限制
- 一条可参考示例

优先在 `Doc/`、`Docs/`、`项目文档/` 下创建；若不存在，则在项目根目录创建 `software-copyright/`。

输出文件：
- `software-copyright/基本信息.md`
- `software-copyright/程序鉴别材料.md`
- `software-copyright/文档鉴别材料.md`（按 [assets/说明文档模板.md](assets/说明文档模板.md) 生成）

## 关键检查清单

### 前置与附加证明

- 著作权人是否已在版权中心**实名认证**（<https://register.ccopyright.com.cn/login.html>）
- 合作/委托/前期开发/非职务/上级法人证明：情形触发时材料是否齐全、签章是否合规

### 鉴别材料

- 软件名称是否在全部材料中一致
- 主要功能描述是否在 500-1300 字
- 程序材料：60 页、每页 50 行、末页 ≥40 行且模块完整、页眉版本号、右上页码、无行号
- 文档材料标题（软件全称）是否在 PDF 中**居中**显示
- 文档材料每张截图下方是否有**居中图注**（`图N-简述`，全文连续编号）
- 文档材料是否按说明文档模板两章结构编写、是否删除注意事项
- 文档材料：图文结合、从登录开始（如有）、页数 **>7 且 ≤60**（建议 13–20 页）
- PDF 是否清晰可读，大小是否建议范围内

## 异常处理

- 信息严重不足：先输出“最小可执行清单”，再请求补充。
- 无法直接生成合规 PDF：在 `README.md` 输出手动生成步骤（页眉、页脚、转换、命名规范）。
- 多版本材料冲突：先列差异，再让用户二选一确认，不自行假设。
