# 软著申请材料审查清单（详细）

主流程以 `SKILL.md` 为准。进入**审核模式**时必须逐项对照本清单；符合触发条件的附加材料**缺一不可**。

---

## 前置条件：实名认证

著作权人需**先在版权中心完成实名认证**，否则无法提交申请。

- 认证网址：<https://register.ccopyright.com.cn/login.html>

审核时首先确认：申请主体（自然人/企业/事业单位等）是否已完成实名认证。

---

## 一、附加证明文件（按情形触发）

> **注意事项**：若软件申请情形符合下列 1–4 任一条，则**必须**提供对应材料；缺少则**无法申请**。

### 1. 合作开发协议

| 触发条件 | 材料要求 |
|----------|----------|
| 著作权人为**多人**，开发方式选择**合作开发** | 提交**合作协议** |

签章要求：
- **自然人**：全部著作权人**签字**
- **企业、事业单位等**：**加盖公章**

### 2. 委托开发协议

| 触发条件 | 材料要求 |
|----------|----------|
| 开发方式为**委托开发** | 提交**委托开发协议** |

签章要求：
- **自然人**：甲乙双方**签字**
- **企业、事业单位等**：**加盖公章**

### 3. 前期开发说明

以下**任一**情形均须提交**前期开发说明**：

| 情形 | 说明 |
|------|------|
| （1） | 公司**成立日期**与软件**开发完成日期**不足 **1 个月** |
| （2） | 软件开发完成日期距公司成立日期不足 **半年**，且**程序量较大**（可按 **1 个月 1 万行**估算是否「较大」） |

材料要求：
- 开发人员**不少于 2 位**
- 说明须**公司加盖公章**
- 开发人员**签字**
- 附全部开发人员：**身份证人像面**、**身份证国徽面**、**签名**

### 4. 非职务开发声明

| 适用 | 不适用 |
|------|--------|
| **企业、事业单位、机关**与**自然人**合作开发情形 | 著作权人**全是自然人**时不需要 |

签章要求：
- **自然人**签字
- **单位负责人**签字
- **单位盖章**

模版请按实际合作开发情形参考官方/实务模板填写。

### 5. 上级法人单位证明

| 触发条件 | 材料要求 |
|----------|----------|
| 著作权人为 **×××分公司**（分支机构） | 提供**上级法人单位证明** |

审核要点：
- 签章必须**真实有效**，不得 PS 或伪造
- 证明中的**公司名称**须与**著作权人**完全一致

## 二、程序鉴别材料（源码文档）

| 项目 | 要求 |
|------|------|
| 页数 | **60 页**（连续前 30 + 后 30） |
| 每页行数 | 第 1–59 页每页 **50 行**有效代码 |
| 最后一页 | **至少 40 行**，并以**完整模块**结尾 |
| 页眉 | 居中：**软件全称 + 版本号** |
| 页码 | **右上角** |
| 行号 | **不需要**行号 |
| 软件名称 | 与申请表、文档材料**完全一致** |

生成时须调用 `scripts/generate_program_pdf.py`，详见 `SKILL.md`。

---

## 三、文档鉴别材料（操作文档 / 说明文档）

| 项目 | 要求 |
|------|------|
| 形式 | **图文结合**，说明软件功能与操作 |
| 截图 | 清晰、完整、跳转**连贯** |
| 流程顺序 | **从登录界面开始**（有登录时）；功能界面按使用顺序依次介绍 |
| 页数 | 硬性：**>7 页**且 **≤60 页**；实务建议 **13–20 页** |
| 标题 | 文首软件全称**居中** |
| 页眉 | 居中：**软件全称 + 版本号** |
| 页码 | **右上角** |
| 模块格式 | 每模块含五项：模块、模块章节、模块描述、模块使用说明、模块截图 |
| 一页一模块 | 五项在 Word/PDF 中**同页占满一页**，模块结束后分页；截图**居中**，过大则缩小至合适尺寸 |
| 图文排版 | 模块使用说明**紧接**截图；避免撑页或整页几乎全白（**不需要**图片下图注） |
| 结构 | 按 [说明文档模板](../assets/说明文档模板.md) 两章结构 |
| 提交前 | 删除模板文首「注意事项」 |

---

## 四、审核模式执行顺序

1. **实名认证**是否已完成
2. 根据申请情形判断 **1–5 附加证明**是否触发 → 逐项核查是否齐全、签章是否符合要求
3. **基本信息**字段与字数、名称一致性
4. **程序鉴别材料**格式与页数
5. **文档鉴别材料**结构、截图、页数
6. 输出问题清单（高/中/低）与补正建议
7. 结论：**通过** / **需修改** / **不通过**
