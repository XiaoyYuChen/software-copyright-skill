# 文档鉴别材料模板说明（详细）

主流程以 `SKILL.md` 为准。生成**文档鉴别材料**时必须基于 skill 内置模板，禁止自由发挥结构。

## 模板文件

| 文件 | 用途 |
|------|------|
| [assets/说明文档模板.doc](../assets/说明文档模板.doc) | 官方 Word 模板（结构与排版参考） |
| [assets/说明文档模板.md](../assets/说明文档模板.md) | Agent 生成用的 Markdown 骨架（占位符版） |

## 模板原文注意事项（生成后须删除）

正式 PDF 中**不得保留**以下内容：

1. 《说明文档》需含有详细的软件操作步骤截图及对应的文字说明（没有操作界面截图则需提供流程图）；界面截图应清晰完整，**有登录界面时从登录界面开始介绍**；无登录可不添加；保证流程连贯。
2. 格式参考模板样式填充内容。
3. 内容需**超过 7 页**，保证内容丰富和完整，**不超过 60 页**。
4. 填充完成后删除注意事项。

## 页数要求（硬性）

| 条件 | 处理 |
|------|------|
| PDF 页数 ≤7 | **不合格**：增加第二章功能模块数量、每模块操作步骤与截图说明，直至 **>7 页** |
| PDF 页数 8–60 | 合格 |
| PDF 页数 >60 | **不合格**：合并或删减次要模块说明，精简至 **≤60 页** |

页数指删除注意事项、插入截图后导出的最终 PDF 页数（含页眉页脚）。

## 文档结构（必须遵守）

### 封面标题

- 第一行：**软件全称**（与申请表、程序鉴别材料页眉一致）
- **PDF 中标题须居中显示**（段落居中对齐，勿左对齐或靠左缩进）

### 第一章 系统描述

| 小节 | 内容要求 |
|------|----------|
| 1.1 系统价值 | 说明软件对用户的帮助、应用场景、实用价值；结合项目 README、主要功能撰写 |
| 1.2 系统功能描述 | 概述功能模块划分，不必逐步操作 |

### 第二章 系统模块介绍

每个功能模块在 Word/PDF 中**独占一页**，且必须包含以下 **5 项**（顺序固定）：

| 序号 | 字段 | 说明 |
|------|------|------|
| 1 | **模块** | 模块名称（标题，如「模块：登录」） |
| 2 | **模块章节** | 章节归属（如「第二章 系统模块介绍 · 2.1 登录」） |
| 3 | **模块描述** | 功能作用、场景、价值（2–4 句，控制篇幅） |
| 4 | **模块使用说明** | 操作步骤、路径、注意点（**紧接**下方截图） |
| 5 | **模块截图** | 运行界面截图，**居中**显示 |

**一页一模块（硬性）**

- 上述 5 项排版后，在 Word/PDF 中应**正好占满一页**（含页眉页脚范围内的正文区）
- 本模块结束后插入**分页符**（Word：Ctrl+Enter），下一模块从新页开始
- 总页数仍须 **>7 且 ≤60**（第一章 + 各模块页合计）

## 图文排版细则（必须）

| 规则 | 要求 |
|------|------|
| 一页一模块 | 5 项内容同页，不得把「模块截图」拆到下一页 |
| 文字与配图 | **模块使用说明**段落后**紧接**模块截图，中间不换页 |
| 截图居中 | 图片在页面中**水平居中**；Markdown/HTML 用 `align="center"` 或 `width="85%"` |
| 图片尺寸 | 若截图过大导致撑破一页，在 Word 中**缩小**至合适尺寸（建议宽度为页面可排印区域的 **80–90%**），保持比例并居中 |
| 避免留白 | 文字过短或图片过小会导致页内大量空白，应增补说明或调大截图 |

Word 排版提示：
- 图片：右键 → 大小和位置 → 勾选「锁定纵横比」→ 宽度约 14–16 cm（A4）或页面宽 85%
- 图片：段落「居中」对齐
- 五项排完后在页末插入分页符；导出 PDF 前逐页预览，确认每模块恰好一页

**最低模块建议：**

- 有登录：**登录** → **主界面** → 各核心业务功能（不少于 3 个）
- 无登录：**主界面/首页** → 各核心业务功能

功能数量按项目实际裁剪或扩展，须覆盖申请表「软件主要功能」要点。

## 排版与页眉（与程序鉴别材料一致）

- **文首标题**（软件全称）：**居中**
- 页眉居中：**软件全称 + 版本号**（如 `某某管理系统V1.0`）
- 页眉右上：**第 N 页**
- 图文结合，截图清晰、尺寸适中

## 输出文件

写入目标项目 `software-copyright/`：

| 文件 | 说明 |
|------|------|
| `文档鉴别材料.md` | 按模板填充的 Markdown（含截图路径占位） |
| `screenshots/` | 运行截图目录（Agent 指引用户补充或由项目截图导出） |
| `文档鉴别材料.docx` / `.doc` | 可选：基于 Word 模板排版 |
| `文档鉴别材料.pdf` | **最终提交件** |

## 生成流程（Agent）

1. 读取 [assets/说明文档模板.md](../assets/说明文档模板.md) 与项目 README、路由/页面结构。
2. 自动识别功能模块列表，替换 `{{功能N名称}}` 等占位符。
3. 生成 `software-copyright/文档鉴别材料.md`；列出待补截图清单（路径 + 对应功能）。
4. 校验：导出 PDF 页数 **>7 且 ≤60**；不足则扩充模块与截图说明，超出则精简；名称与版本号全链路一致；有登录则含登录章节。
5. 指引用户插入截图并导出 PDF（**标题居中**、**每模块五项一页**、**截图居中且尺寸合适**、页眉页脚按上文设置；逐页检查无撑页、无大面积空白）。

## 审核要点

- 是否按模板两章结构编写
- 是否删除模板注意事项
- 每模块是否包含**五项**（模块、模块章节、模块描述、模块使用说明、模块截图）
- 每模块在 PDF 中是否**独占一页**、模块结束后是否分页
- 模块使用说明是否**紧接**截图；截图是否**居中**、尺寸是否合适
- 是否存在撑破多页或整页几乎全白
- 截图是否清晰、流程是否连贯、是否从登录开始（如有）
- 页数是否 **>7 且 ≤60**（超过 7 页、不超过 60 页，内容丰富完整）
- 文首标题（软件全称）是否居中
- 页眉是否为「软件全称+版本号」、右上角是否有页码
